## How can I ensure that my code, program, or software is functional and secure?
To ensure that my code, program, or software is functional and secure will require the development of strong test cases to test all aspects of the developed software. It is also important to look at the test cases through the lens of someone who only knows the requirements and not how the code works. By taking this approach, I will ensure that dveloped test cases test all positive and negative results of the software. Any issue found can be fixed, and if needed, addtional test cases can be created.

## How do I interpret user needs and incorporate them into a program?
To interperet user needs requires review of all documentation and requirements prior to code development. This means reviewing documentation to ensure that it is clear and any ambiguities are called out prior to the start of coding. Development starts at testing the documentation. Once this has been completed, requirements should be mapped out and developed to meet. Tessting should be conducted at each step of the process to ensure a solid final product is delivered to the end user and the stakeholder.

## How do I approach designing software?
I used to approach designing software with a "get it done" attitude. This meant, that I hit all the needs and tried my best to make a secure and sound product. Now that I have completed this course on test development, I have learned the importance of closing loose ends in my developed software and have changed my approach to "get it done right". This involves both development and testing that goes hand in hand to not only provide the best version of the software, but a more robust and secure version as well.
